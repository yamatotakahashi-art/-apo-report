# 【打診】営業報告ツールにおける顧客データ共有方式について

システム部門 ご担当者様

社内の営業報告ツール（Google ログイン制・社内ドメイン限定）の機能拡張にあたり、
顧客データの取り扱いについてご相談です。最も安全な方式をご教示いただきたく、
下記に現状・やりたいこと・確認事項をまとめました。

---

## 1. ツールの概要

- 用途：営業代行業務の **架電結果報告 → Slack 報告文・メール下書きの整形** を補助する社内ツール。
- 認証：Google ログイン（社内ドメイン `marchon.co.jp` 限定、メール検証済みのみ）。役割は staff / intern。
- ホスティング：Vercel（Next.js）。DB：Neon（PostgreSQL）。
- **現状のデータ方針**：
  - エンド顧客データ（会社名・電話・住所・顧客メール・担当者名・架電メモ）は **利用者のブラウザ内（localStorage）のみ** に保持し、サーバ・外部に送信していません。
  - サーバ（Neon）に保存しているのは **設定情報のみ**（ユーザー・役割・テンプレート文面など）。
  - 外部送信は Slack へのフィードバック（バグ/要望）のみで、送信前に顧客情報を伏せ字化しています。

## 2. 取り扱うデータの種類

| 区分 | 例 | 機微度 |
|------|----|--------|
| エンド顧客データ | 送付先の会社名・氏名・電話・住所・メール | 高（最も保護したい） |
| 依頼主（発注元）情報 | 営業を委託してきた取引先のCC用メールアドレス | 中（外部の取引先連絡先） |
| 設定・集計 | 案件名・Slackチャンネル名・テンプレ文面・件数 | 低（非PII） |

## 3. やりたいこと（今回の拡張）

1. **報告履歴のチーム共有**
   - これまで各自のブラウザ内に留めていた報告履歴（会社名等の顧客データを含む）を、**チームで閲覧・共有** したい。
2. **上長への確認依頼（任意・ソフト承認）**
   - 報告内容について、本人の希望で **上長が任意で内容を確認** できる軽い仕組みを付けたい（必須承認ではない）。
   - 上長が内容を見る＝顧客データを上長と共有することになります。
3. **依頼主（発注元）CCの設定保存**
   - 案件ごとに固定の依頼主CCアドレスを、ツールの設定としてサーバ（Neon）に保存したい。

## 4. ご相談・確認したいこと

1. 上記1・2のように **顧客データ（会社名等）をサーバに保存・共有** する場合、当社として承認される方式は何でしょうか。
   - 例：暗号化（保存時/通信時）、アクセス制御（社内ドメイン＋役割）、保持期間・削除運用、監査ログ、保存先（Neon でよいか／別の承認済み基盤を使うべきか）。
2. **データの保存場所（リージョン/事業者）** に制約はありますか（国内限定の要否、利用可能なクラウド事業者など）。
3. 上長確認において、**顧客データそのものを見せてよい範囲** はどこまでですか。
   - もし顧客データの共有が不可なら、**非PIIメタデータ（案件・成果区分・日時のみ）** に絞る案で問題ないでしょうか。
4. 依頼主（発注元）CCアドレスを設定として保存することに問題はありますか。
5. その他、社内規程・個人情報保護方針で留意すべき点があればご教示ください。

## 5. 暫定運用（ご回答までの間）

ご回答をいただくまでは、安全側に倒して以下で進めます。
- 報告履歴の **顧客データ明細は各自のブラウザ内のみ**（サーバには非PIIの件数集計だけ）。
- 上長確認依頼は **非PIIメタデータのみ** を共有（顧客データ本体は共有しない）。

ご多用のところ恐れ入りますが、ご確認のほどよろしくお願いいたします。
